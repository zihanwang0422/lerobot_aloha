## 服务器 GPU 环境配置

`pip install -e ".[feetech]"` 会自动安装 CPU 版 PyTorch，**需要手动替换成 CUDA 版本**。

### 1. 确认 CUDA 版本

```bash
nvidia-smi        # 查看驱动支持的最高 CUDA 版本
nvcc --version    # 查看已安装的 CUDA 工具链版本
```

### 2. 安装对应 CUDA 版本的 PyTorch

前往 [https://pytorch.org/get-started/locally/](https://pytorch.org/get-started/locally/) 选择对应版本，常见命令：

```bash
# CUDA 12.4（A100 / H100 服务器推荐）
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu124

# CUDA 12.1
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121

# CUDA 11.8
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118
```

### 3. 验证 CUDA 可用

```python
import torch
print(torch.cuda.is_available())       # 应输出 True
print(torch.cuda.device_count())       # 应输出 GPU 数量（如 4）
print(torch.cuda.get_device_name(0))   # 输出 GPU 型号
```

# 双臂主从遥操作 + 数据采集 + 训练 ACT (Aloha)

## 整体架构

```
主臂(Leader) × 2  →  bi_so_leader（遥操作器）
                         ↓ 关节角度
从臂(Follower) × 2 ←  bi_so_follower（机器人）
                         ↓ 录制
                     LeRobotDataset（数据集）
                         ↓ 训练
                     ACT Policy（双臂 Aloha）
```

动作维度：双臂共 **14 维** = 左臂 6 关节 + 1 夹爪 + 右臂 6 关节 + 1 夹爪

---

## 第一步：串口权限 & 固定端口绑定

每次上电后 `/dev/ttyACM*` 编号可能变化，通过 udev 规则将四条臂绑定到固定路径：

| 固定路径 | 臂 |
|---|---|
| `/dev/robot_leader_left`    | 左主臂 |
| `/dev/robot_leader_right`   | 右主臂 |
| `/dev/robot_follower_left`  | 左从臂 |
| `/dev/robot_follower_right` | 右从臂 |

### 1.1 识别各臂序列号（首次配置执行一次）

```bash
bash identify_arms.sh
```

按提示逐一拔插各臂 USB 线，脚本自动生成 `99-lerobot-arms.rules`。

### 1.2 安装 udev 规则

```bash
sudo cp 99-lerobot-arms.rules /etc/udev/rules.d/
sudo udevadm control --reload-rules && sudo udevadm trigger

# 验证绑定
ls -la /dev/robot_*
```

### 1.3 串口权限

```bash
sudo usermod -aG dialout $USER   # 重新登录后永久生效
# 或临时授权：
sudo chmod 666 /dev/robot_*
```

### 1.4 查找摄像头索引

```bash
lerobot-find-cameras opencv
```

```bash
  }' \
  --robot.right_arm_config.cameras='{
    top: {type: opencv, index_or_path: "/dev/video0", width: 640, height: 480, fps: 30, fourcc: "MJPG"}
  }' \
  --teleop.type=bi_so_leader \
  --teleop.id=bimanual_leader \
  --teleop.left_arm_config.port=/dev/robot_leader_left \
  --teleop.right_arm_config.port=/dev/robot_leader_right \
  --display_data=true \
  --dataset.repo_id=${HF_USER}/bimanual_test \
  --dataset.num_episodes=50 \
  --dataset.single_task="双臂协作——将红色方块从左臂传递到右臂" \
  --dataset.push_to_hub=true \
  --dataset.episode_time_s=30 \
  --dataset.reset_time_s=15
```

录制按键：
- `→`（右箭头）— 保存当前 episode，进入下一个
- `←`（左箭头）— 丢弃当前 episode，重新录制
- `ESC` — 保存并退出

> **摄像头键名必须与采集时一致**（`top`），训练和评估时都要保持相同命名，否则会报 `mean is infinity` 错误。

## 第六步：训练双臂 ACT 策略

### 数据放置位置

LeRobot 数据集默认缓存路径为：

```
~/.cache/huggingface/lerobot/<用户名>/<数据集名>/
```

在采集时加了 `--dataset.push_to_hub=false`，数据就存在这里。  
**服务器上训练时**，将整个数据集目录拷贝到服务器的高速 SSD，例如：

```bash
# 在采集机上打包
tar -czf bimanual_test.tar.gz \
  ~/.cache/huggingface/lerobot/seeedstudio123/bimanual_test

# 传输到服务器
scp bimanual_test.tar.gz user@server:/data/lerobot/

# 服务器上解压（保持目录结构）
ssh user@server
mkdir -p /data/lerobot
tar -xzf /data/lerobot/bimanual_test.tar.gz -C /data/lerobot/
# 最终路径：/data/lerobot/seeedstudio123/bimanual_test/
```

训练时通过 `--dataset.root` 指定数据根目录，无需从 Hub 下载：

```bash
--dataset.root=/data/lerobot
```

---

### 单卡训练（单 GPU）

```bash
lerobot-train \
  --dataset.repo_id=seeedstudio123/bimanual_test \
  --dataset.root=/data/lerobot \
  --policy.type=act \
  --output_dir=outputs/train/act_bimanual \
  --job_name=act_bimanual \
  --policy.device=cuda \
  --wandb.enable=false \
  --policy.push_to_hub=false \
  --steps=300000
```

---

### 多卡训练（使用 GPU 0,1,2,3）

LeRobot 通过 HuggingFace Accelerate 支持多卡 DDP 训练，使用 `accelerate launch` 替换 `lerobot-train`：

```bash
# 安装 accelerate（已安装可跳过）
pip install accelerate

# 指定使用前四卡 0,1,2,3
CUDA_VISIBLE_DEVICES=0,1,2,3 accelerate launch \
  --multi_gpu \
  --num_processes=4 \
  --mixed_precision=bf16 \
  $(which lerobot-train) \
  --dataset.repo_id=seeedstudio123/bimanual_test \
  --dataset.root=/data/lerobot \
  --policy.type=act \
  --output_dir=outputs/train/act_bimanual \
  --job_name=act_bimanual \
  --wandb.enable=false \
  --policy.push_to_hub=false \
  --batch_size=8 \
  --steps=75000
```

> **多卡超参数调整说明：**
>
> - 有效 batch size = `batch_size × num_gpus`，例如 `8 × 4 = 32`
> - 建议按比例缩减 steps：原 `300000 steps（batch=8）` → `75000 steps（batch=8×4张卡）`
> - 或线性放大学习率：原 `lr=1e-4` → `4e-4`（4 卡线性缩放）
> - `--mixed_precision=bf16` 推荐用于 A100/H100，RTX 系列用 `fp16`

> **说明：** 双臂动作维度为 14，数据集中已自动记录，`policy.type=act` 会从数据集元信息中自动推断输入输出维度，无需手动指定。

## 第七步：评估双臂策略

```bash
lerobot-record \
  --robot.type=bi_so_follower \
  --robot.id=bimanual_follower \
  --robot.left_arm_config.port=/dev/robot_follower_left \
  --robot.right_arm_config.port=/dev/robot_follower_right \
  --robot.left_arm_config.cameras='{
    left: {type: opencv, index_or_path: "/dev/video2", width: 640, height: 480, fps: 30, fourcc: "MJPG"}
  }' \
  --robot.right_arm_config.cameras='{
    top: {type: opencv, index_or_path: "/dev/video0", width: 640, height: 480, fps: 30, fourcc: "MJPG"}
  }' \
  --display_data=false \
  --dataset.repo_id=seeedstudio123/eval_bimanual \
  --dataset.single_task="双臂协作——将红色方块从左臂传递到右臂" \
  --dataset.push_to_hub=false \
  --policy.path=outputs/train/act_bimanual/checkpoints/last/pretrained_model
```

> - `--policy.path` 指向训练输出的最终权重目录。
> - `dataset.repo_id` 建议以 `eval_` 开头，系统会自动分目录存储评估录像。
> - 若提示 `File exists`，先删除对应的 `eval_` 目录再重新运行。
